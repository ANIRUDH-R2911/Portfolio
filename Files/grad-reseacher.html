01 · Overview

What Was This Work About?

Autonomous driving stacks depend on object detectors to reliably identify vehicles and pedestrians in real time. But how do modern standalone YOLO models actually compare against the detector already embedded inside a production-grade framework like Autoware? That question — simple in theory, messy in practice — was the core problem this research set out to answer.

A Python-based benchmarking framework was built from scratch to fairly evaluate four YOLO-based detectors — YOLOv8 (fine-tuned), YOLOv9, YOLO12, and Autoware's built-in YOLOX — on the same 1,900-image subset of the Waymo Open Dataset, under identical conditions and standardized metrics.

The challenge wasn't just running inference. Each model lives in a different ecosystem: YOLOv8/9/12 run through Python inference pipelines, while Autoware's YOLOX operates exclusively inside a ROS2 perception stack. Making them comparable required a shared annotation format, a custom ROS2 publisher node, and a unified evaluation pipeline built around JSON ground-truth files.

04 · Methodology

How the Framework Was Built

01

Dataset Preparation

Extracted 10,000 front-camera images from Waymo sequences. Filtered for frames containing Person or Vehicle instances. Split into 8,000 training + 1,900 fixed validation images.

02

Annotation Conversion

Waymo's 3D labels were projected and converted into 2D YOLO-format bounding boxes. A custom script exported all annotations as structured JSON ground-truth files for unified evaluation.

03

ROS2 Pipeline for Autoware

A dedicated ROS2 image publisher node streamed the 1,900 validation frames into Autoware's /sensing/camera topic, simulating a live onboard camera. Detections were captured from the /rois0 topic and saved as CSV.

04

Output Harmonization

Autoware's CSV outputs and YOLO model predictions were all converted to an identical JSON format with matching keys, class mappings, and bounding box normalization for per-image comparison.

05

Evaluation

Computed mAP@50, mAP@50:95, Precision, Recall, and F1-score per model and per class (Person, Vehicle) across the fixed 1,900-image validation set.

05 · Results

Performance Comparison

All four models were evaluated on the same 1,900-image Waymo validation split. YOLOv8 (fine-tuned) outperformed every other model across all metrics, with the most dramatic gap on pedestrian detection.

Model mAP@50 mAP@50:95 Precision Recall F1
YOLOv9 0.47 0.42 0.35 0.38 0.36
YOLO12 0.48 0.43 0.35 0.37 0.36
YOLOX (Autoware) 0.54 0.42 0.44 0.48 0.46
YOLOv8 (Fine-tuned) Best 0.65 0.45 0.60 0.64 0.62

Pedestrian detection told an even clearer story. YOLOv8 achieved 0.85 mAP@50 and 0.86 recall on the Person class — nearly double the pre-trained models (0.35) and well above Autoware's YOLOX (0.39). Pedestrians are inherently harder to detect due to small size, occlusion, and appearance variability, making domain adaptation especially impactful for this class.

What I Took Away

The most important insight wasn't the number — it was understanding why the gap existed. Pre-trained models that work well on COCO struggle on driving-specific imagery because the visual patterns are fundamentally different. Building the entire pipeline from annotation conversion to ROS2 integration taught me how much friction lies between "a model that works in a notebook" and "a model that works in a system." The benchmarking framework itself is the contribution — not just the final mAP.
